# Лабораторная работа: фильтрация Salt-and-Pepper шума на GPU

## Описание задачи

В работе нужно удалить шум типа **Salt-and-Pepper** с grayscale-изображения в
формате BMP. Такой шум выглядит как случайные черные и белые пиксели:

- `0` - pepper, черная точка;
- `255` - salt, белая точка.

По условию нужно применить **9-точечный медианный фильтр** к изображению размера
`M x N`, сохранить результат в BMP и вывести время обработки на GPU. Для краев
изображения недостающие соседние пиксели берутся из ближайших доступных пикселей
через clamp.

## Описание реализации

### 1. Чтение и запись BMP

Чтобы проект легко запускался на Windows без внешних библиотек, в `main.cu`
реализованы простые функции:

- `readBmpGray(...)` - читает BMP и возвращает grayscale-изображение;
- `writeBmpGray8(...)` - сохраняет результат как 8-bit grayscale BMP.

Основной входной формат - 8-bit grayscale BMP. Для удобства также поддержано
чтение 24-bit и 32-bit BMP: такие изображения автоматически переводятся в
grayscale.

### 2. CPU-фильтр

CPU-версия нужна для проверки корректности и сравнения времени.

Для каждого пикселя берется окно `3 x 3`:

```text
p00 p01 p02
p10 p11 p12
p20 p21 p22
```

Значения сортируются простым bubble sort, после чего в результат записывается
средний элемент с индексом `4`. Это и есть медиана для 9 значений.

### 3. GPU-фильтр

GPU-версия реализована в `medianFilterTextureKernel`.

Что делает каждый CUDA-поток:

- отвечает за один пиксель выходного изображения;
- берет исходный пиксель и 8 соседей через texture memory;
- для краев использует ближайшие пиксели;
- сортирует 9 значений;
- записывает медиану в выходной массив.

По условию CUDA-реализация должна использовать texture memory. В проекте для
этого создается `cudaTextureObject_t`, а чтение пикселей внутри kernel идет через
`tex2D<unsigned char>(...)`.

### Что именно распараллелено и почему

Распараллелена обработка пикселей. Медианное значение для каждого пикселя можно
вычислять независимо от остальных, потому что фильтр читает только исходное
изображение и пишет только свой пиксель результата.

Это хорошо ложится на CUDA:

- один поток обрабатывает один пиксель;
- блоки потоков независимо покрывают разные участки изображения;
- texture memory удобна для пространственного доступа к соседним пикселям.

## Особенности измерения времени

Программа выводит:

- среднее время CPU-фильтра;
- среднее полное время GPU-запуска;
- среднее время только CUDA kernel;
- ускорение GPU относительно CPU;
- количество отличающихся пикселей при сравнении CPU и GPU результата.

В `GPU total` входит копирование входного изображения на устройство, запуск
kernel и копирование результата обратно. Выделение GPU-памяти и создание
texture object в основной замер не входят. Перед измерением GPU выполняется
один warm-up запуск.

## Сборка

Требования:

- NVIDIA GPU;
- установленный CUDA Toolkit;
- компилятор `nvcc`;
- CMake;
- на Windows: Visual Studio 2022 с C++ workload.

Сборка через CMake:

```powershell
cd lab4_salt_and_pepper
cmake -S . -B build
cmake --build build --config Release
```

После сборки исполняемый файл будет лежать прямо в `build`:

```powershell
.\build\saltpepper.exe
```

Также оставлен простой `Makefile`:

```bash
make run
```

## Запуск

Запуск по умолчанию:

```powershell
.\build\saltpepper.exe
```

По умолчанию программа читает:

```text
data/noisy_256.bmp
```

и пишет:

```text
filtered.bmp
```

Запуск с явным входом и выходом:

```powershell
.\build\saltpepper.exe --input data\noisy_256.bmp --output filtered.bmp
```

Запуск с фиксированным числом повторов:

```powershell
.\build\saltpepper.exe --input data\noisy_256.bmp --output filtered.bmp --repeats 50
```

Генерация своего тестового BMP через саму программу:

```powershell
.\build\saltpepper.exe --generate-test data\noisy_512.bmp --width 512 --height 512 --noise 0.08
.\build\saltpepper.exe --input data\noisy_512.bmp --output filtered_512.bmp
```

## Тестовые данные

В проект уже добавлен тестовый файл:

```text
data/noisy_256.bmp
```

Это grayscale BMP размером `256 x 256` с искусственно добавленным
Salt-and-Pepper шумом. Поэтому после сборки можно сразу запускать:

```powershell
.\build\saltpepper.exe

Также есть Python-скрипт без внешних зависимостей:

```

## Формат результата

Программа выводит информацию о GPU и Markdown-таблицу:

```text
GPU: NVIDIA ...
SM count: ...
CUDA block: 16x16
Max threads per block: ...

| Input | Size | Repeats | CPU, ms | GPU total, ms | GPU kernel, ms | Speedup total | Speedup kernel | Mismatched pixels | Max abs diff | Output |
```

Что ожидаем увидеть:

- `Mismatched pixels = 0`;
- `Max abs diff = 0`;
- в `Output` будет путь к отфильтрованному BMP;
- на маленьких изображениях GPU может не дать большого ускорения из-за накладных
  расходов;
- на больших изображениях GPU обычно становится выгоднее, потому что каждый
  пиксель обрабатывается независимо.

## Вывод

В лабораторной реализован медианный фильтр для удаления Salt-and-Pepper шума.
Главная идея распараллеливания состоит в независимой обработке пикселей:
каждый CUDA-поток берет окно `3 x 3`, сортирует 9 значений и записывает медиану.
Texture memory используется для чтения исходного изображения, что соответствует
условию задания и удобно для доступа к соседним пикселям.
