# Лабораторная работа: вычисление числа pi методом Монте-Карло на CPU и GPU

## Описание задачи

В работе нужно вычислить приближенное значение числа pi методом Монте-Карло
двумя способами:

1. Последовательно на CPU.
2. Параллельно на GPU с использованием CUDA и CURAND device API.

Метод использует случайные точки в квадрате `[0, 1] x [0, 1]`. Для каждой
точки проверяется, попала ли она внутрь четверти единичного круга:

```text
x^2 + y^2 <= 1
```

Если `inside` - количество попавших точек, а `N` - общее количество точек, то:

```text
pi ~= 4 * inside / N
```

Чем больше `N`, тем ближе оценка к настоящему значению pi.

## Описание реализации

### 1. Вычисление на CPU

CPU-версия генерирует пары координат `(x, y)` через `std::mt19937_64` и
`std::uniform_real_distribution<double>`.

Для каждой точки выполняется обычная последовательная проверка:

```text
if x^2 + y^2 <= 1, то точка считается попавшей в круг
```

После этого считается приближение:

```text
pi = 4 * inside / N
```

CPU-результат нужен как базовый вариант для сравнения времени выполнения.

### 2. Вычисление на GPU

GPU-версия реализована через CUDA kernel-функции.

В `countInsideCircleKernel` каждый CUDA-поток:

- инициализирует собственное состояние генератора `curandStatePhilox4_32_10_t`;
- генерирует случайные координаты через `curand_uniform`;
- проверяет попадание точки в четверть круга;
- считает локальное количество попавших точек;
- участвует в редукции внутри блока через `shared memory`.

Каждый блок записывает один частичный результат. Затем
`reduceCountsKernel` несколько раз суммирует частичные результаты на GPU, пока
не останется одно число - итоговое количество точек внутри круга. Финальное
значение pi вычисляется в `computePiKernel`, то есть GPU-ветка выполняет и
подсчет попаданий, и формулу `4 * inside / N` на устройстве.

Итоговая формула такая же:

```text
pi = 4 * inside / N
```

### Что именно распараллелено и почему

На GPU распараллелена обработка независимых случайных точек. Для каждой точки
нужно только сгенерировать `(x, y)` и проверить условие `x^2 + y^2 <= 1`, поэтому
разные потоки могут делать это одновременно без конфликтов.

После локальных подсчетов используется параллельная редукция: сначала внутри
каждого блока, затем между блоками. Такой подход похож на редукцию из
`lab2_vectorsum`, но вместо сложения входного массива сначала генерируются точки
и считается количество попаданий.

CPU и GPU используют разные генераторы случайных чисел, поэтому значения pi
могут немного отличаться. Это нормально для метода Монте-Карло.

## Особенности измерения времени

- Для CPU измеряется среднее время одного запуска генерации точек и подсчета.
- Для GPU измеряется среднее время одного запуска, включающее:
  - генерацию случайных чисел через CURAND на устройстве;
  - проверку попадания точек;
  - редукцию на GPU;
  - вычисление финального значения pi на GPU;
  - копирование финального счетчика и значения pi `Device -> Host`.
- Выделение и освобождение GPU-памяти не входит в основной замер.
- Перед измерениями выполняется один warm-up запуск GPU.

## Сборка

Требования:

- NVIDIA GPU;
- установленный CUDA Toolkit;
- компилятор `nvcc`;
- CMake;
- на Windows: Visual Studio 2022 с C++ workload.

Сборка через CMake:

```powershell
cd lab3_pi_monte_carlo
cmake -S . -B build
cmake --build build --config Release
```

После сборки исполняемый файл будет лежать прямо в `build`, поэтому запуск
выглядит так:

```powershell
.\build\pimontecarlo.exe
```

Также оставлен простой `Makefile`:

```bash
make run
```

## Запуск

Запуск набора экспериментов по умолчанию:

```powershell
.\build\pimontecarlo.exe
```

По умолчанию используются размеры:

- `10 000`
- `100 000`
- `1 000 000`
- `5 000 000`
- `10 000 000`

Быстрая проверка после сборки:

```powershell
.\build\pimontecarlo.exe --repeats 1 1000 10000
```

Запуск с собственными размерами:

```powershell
.\build\pimontecarlo.exe 10000 100000 1000000 10000000
```

Запуск с фиксированным числом повторов:

```powershell
.\build\pimontecarlo.exe --repeats 10 100000 1000000 10000000
```

Запуск с фиксированным seed и размером CUDA-блока:

```powershell
.\build\pimontecarlo.exe --seed 42 --threads 256 --repeats 5 1000000
```

Параметр `--threads` должен быть степенью двойки от `32` до `1024`.

## Формат результата

Программа выводит информацию о GPU и Markdown-таблицу:

```text
GPU: NVIDIA ...
SM count: ...
Warp size: ...
Threads per block: ...
Max threads per block: ...

| Points | Repeats | CPU inside | GPU inside | CPU pi | GPU pi | Abs diff | CPU, ms | GPU total, ms | GPU kernel, ms | Speedup total | Speedup kernel |
```

Столбцы:

- `Points` - количество случайных точек `N`;
- `CPU inside` и `GPU inside` - сколько точек попало в четверть круга;
- `CPU pi` и `GPU pi` - вычисленное приближение pi;
- `Abs diff` - абсолютная разница между CPU и GPU оценками;
- `CPU, ms` - среднее время CPU;
- `GPU total, ms` - среднее полное время GPU-запуска;
- `GPU kernel, ms` - среднее время CUDA kernel-функций;
- `Speedup total` - ускорение по полному GPU-времени;
- `Speedup kernel` - ускорение только по времени kernel-функций.

## Вывод

В лабораторной реализовано вычисление числа pi вероятностным методом
Монте-Карло. Основная идея распараллеливания состоит в том, что проверка каждой
случайной точки независима от остальных. Поэтому GPU может одновременно
обрабатывать большое количество точек, а затем быстро объединять частичные
счетчики через редукцию.

На малых `N` GPU может проигрывать CPU из-за накладных расходов на запуск
kernel-функций и CURAND. На больших `N` ожидается ускорение, потому что объем
независимой работы становится достаточно большим для эффективной загрузки GPU.
